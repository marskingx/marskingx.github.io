請扮演一位資深的 UI/UX 設計師和前端工程師。

我的任務是為我的個人理財部落格（風格類似 lazytoberich.com.tw）設計一個全網站使用的電子報訂閱彈出視窗 (Popup)。

請根據以下需求，提供可以直接使用的 HTML 和 CSS 程式碼。

## 1. 核心目標與風格
- **主要目標**: 免費提供電子報的Email 訂閱。
- **視覺風格**: 專業、現代、極簡。
- **色調**: 主要參考目前網站css的主色調，基本是黑、白、灰做明暗的調整。

## 2. 結構與內容
- **圖片**: 在左側或頂部（根據桌面/手機版調整）放置一張專業、抽象符合「| Podcast | 理財 | 投資 | 生活 | 閱讀 | Mindfulness |」形象的圖片
- **標題**: "免費訂閱電子報"
- **副標題**: "立即訂閱免費電子報，即可獲得您第一手理財資訊！"
- **輸入欄位**: 一個欄位，用於輸入 Email，提示文字為 "請輸入您的電子郵件"。
- **提交按鈕**: 顏色參考網站主色調，文字為「免費訂閱」。
- **關閉方式**: 右上角有一個清晰的 "X" 關閉圖示。

## 3. 行為與互動
- **觸發時機**: 當使用者進入任何頁面，停留 1 秒後，此彈出視窗以淡入 (fade-in) 動畫出現。
- **成功回饋**: 當使用者點擊「立即訂閱」後，彈出視窗的內容應替換為一個成功訊息，標題為「感謝您的訂閱！」，內容為「恭禧你即將成為懶又有錢的人。」
- **響應式設計**:
  - 在桌面版，圖片在左，文字內容在右。
  - 在手機版（寬度小於 768px），圖片在頂部，文字內容在下方。佈局必須是垂直堆疊。

## 4. 技術產出要求
- **HTML**: 請提供結構清晰、語意化的 HTML。
- **CSS**: 請提供封裝好的 CSS，盡量使用 class 命名，避免影響網站全域樣式。請包含淡入動畫和響應式設計的 media query。
- **JavaScript**: 請提供觸發彈出視窗（1秒延遲）和處理訂閱成功後訊息替換的基本 JavaScript 邏輯。

## 1. 技術棧 (Tech Stack)
- **靜態網站生成器**: Hugo
- **電子報服務**: Mailchimp
- **程式碼倉庫 & CI/CD**: GitHub & GitHub Actions

## 2. 自動化工作流程需求 (Workflow Requirements)
1.  **觸發條件**: 當有新的 commit 被 push 到 `main` 分支時，這個工作流程應該被觸發。
2.  **新文章判斷**: 工作流程必須先判斷這次的 commit 是否真的包含一篇「新文章」（例如，在 `content/posts/` 目錄下有新增的 `.md` 檔案）。這麼做是為了避免在修改主題或修正錯字時也寄送通知信。
3.  **內容擷取**: 如果確定有新文章，腳本需要從該文章的 Markdown 檔案中，擷取出**標題 (title)**、**摘要 (summary)** 和**完整的文章永久連結 (permalink)**。
4.  **API 串接**: 腳本接著需要使用 Mailchimp API 來：
    - 建立一個新的行銷活動 (Campaign)。
    - 將擷取到的文章標題和摘要作為信件內容。
    - 將這個行銷活動發送給指定的訂閱者列表 (Audience List)。

## 3. 技術產出要求 (Technical Output Requirements)
1.  **GitHub Actions 設定檔**: 請提供一個完整的、包含註解的 GitHub Actions workflow 檔案 (`.github/workflows/mailchimp_notification.yml`)。
2.  **自動化腳本**: 請提供一個由 workflow 呼叫的腳本（建議使用 Python 或 JavaScript/Node.js），這個腳本負責處理新文章的判斷、內容擷取和 Mailchimp API 的所有互動。
3.  **安全設定**: 請詳細說明如何安全地將 **Mailchimp API Key** 和 **Audience ID** 存放在 GitHub Secrets 中，並在 workflow 和腳本中正確地使用它們。
